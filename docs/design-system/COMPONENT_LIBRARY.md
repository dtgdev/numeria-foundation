# Numeria Component Library

**Version:** 1.0 — Genesis

---

# Purpose

The Component Library defines the reusable building blocks that create every Numeria experience.

Rather than building interfaces from scratch, applications assemble standardized components.

Consistency creates familiarity.

Familiarity reduces cognitive load.

Reduced cognitive load improves learning.

---

# Philosophy

Every component should be:

- Beautiful
- Accessible
- Reusable
- Predictable
- Educational
- Responsive

A component is more than UI.

It is part of the educational experience.

# Core Components

The following components are considered foundational.

---

## Entity Card

Displays a single Entity.

Properties:

- Icon
- Name
- Category
- Description
- Color
- Status
- Related Count

Used in:

- Search
- Explorer
- Studio
- AI Results

---

## Relationship Edge

Visual connection between two Entities.

Properties:

- Source
- Target
- Type
- Strength
- Direction
- Color
- Animation

Used in:

- Universe Explorer
- Knowledge Graph
- Story Builder

---

## Entity Inspector

Displays complete Entity information.

Sections:

- Overview
- Mathematics
- Story
- Relationships
- Educational Purpose
- Canon
- History

---

## Graph Toolbar

Provides graph controls.

Functions:

- Search
- Zoom
- Fit
- Layout
- Filter
- Export
- AI Assistant

---

## Search Bar

Supports natural language search.

Examples:

"Derivative"

"Functions"

"Stories about Limits"

"What teaches Integrals?"

## Knowledge Badge

Displays metadata.

# Interaction Principles

Every component should provide:

Immediate feedback

Clear purpose

Minimal friction

Consistent behavior

Accessible controls

Responsive layout

---

# Component States

Each interactive component supports:

- Default
- Hover
- Focus
- Active
- Selected
- Disabled
- Loading
- Error
- Success

State transitions should be smooth and predictable.

---

# Accessibility

All components must support:

- Keyboard navigation
- Screen readers
- High contrast
- Reduced motion
- Touch devices
- Responsive layouts

Accessibility is required, not optional.

---

# Extensibility

Components should be composable.

Applications should extend components rather than duplicate them.

The Component Library is the single source of truth for Numeria interfaces.

---

# Closing

The Component Library transforms design principles into reusable experiences.

Every screen in Numeria should be built from these shared components, ensuring that every learner encounters a familiar, beautiful, and consistent interface wherever they explore.
